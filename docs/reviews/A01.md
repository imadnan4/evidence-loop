# A01 Review — Platform foundation

## Revision and scope
Reviewed the uncommitted A01 platform/configuration diff after A00. Changed paths are limited to root workspace configuration, `infra/**`, `.github/**`, `packages/config/**`, check scripts, and the API check script.

## Checks
- `pnpm run check:all` — passed.
- `pnpm run compose:smoke` — passed; it created a unique temporary Compose project, provisioned all three object zones, verified authenticated visibility and anonymous list/read denial for `quarantine`, `clean`, and `derived`, then removed containers, volumes, network, and temporary credentials.
- `git diff --check` — passed.
- Container builds and Compose config validation — passed during review.

## Acceptance evidence
- Environment validation is server-only, fails closed, redacts values, rejects browser-public namespaces, and requires `SYNTHETIC_DATA_ONLY=true`.
- PostgreSQL and MinIO bind only to loopback; API/worker have no published host ports; object zones are initialized idempotently with anonymous access disabled.
- The root check now includes the API session state-machine suite.
- CI runs frozen install, workspace checks, image builds, Compose smoke, dependency audit, and a tracked-secret heuristic.
- Documentation is explicit that this is a synthetic local dependency stack, not a deployed API, production environment, or PII-authorized system.
- `infra/**/*.md` is explicitly eligible for version control, so the README operations link is not broken after commit.

## Residual risks
Provider/region, PII authorization, scanner/parser isolation, durable database schema, deployment, backup/restore proof, and full observability remain later gates. `.pi-subagents/` is unrelated runtime output and must remain uncommitted.

## Decision
APPROVED
